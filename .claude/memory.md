# PDF文献元数据提取工具 - 项目记忆

## 📋 项目架构

### 主要功能
1. **批量PDF处理** - 支持多文件夹、递归搜索
2. **智能漏斗策略** - 三级处理流程：
   - 首次扫描：pymupdf4llm处理前3页，判断相关性和文档类型
   - DOI提取：正则→LLM文本→视觉模型三级漏斗
   - 元数据提取：DOI搜索→LLM文本→视觉模型三级漏斗
3. **多数据源支持** - OpenAlex、Crossref、Semantic Scholar
4. **智能分类** - 只处理期刊和相关文献，自动过滤书籍和非相关文献
5. **CSV合并工具** - 独立的GUI工具用于结果合并

### 核心库依赖
- **PyMuPDF (fitz)** - PDF文本提取和页面操作
- **pymupdf4llm** - OCR和markdown转换
- **pypdf** - 页面截取功能
- **LLMEhancer** - 自定义LLM增强模块
  - 使用DeepSeek API进行文本分析
  - 使用Kimi-VL视觉模型进行图像识别
- **pandas** - CSV处理
- **requests** - API调用
- **customtkinter** - GUI界面
- **OpenAI库** - LLM API调用

## 🐛 已解决的重大Bug

## ⚙️ 核心配置参数

### 环境配置
- **CPU**: AMD Ryzen 9 7840HS (8核16线程)
- **内存**: 32GB DDR5
- **系统**: Windows 11 + WSL2 Ubuntu 22.04
- **Python**: 3.8+

### API配置
- **DeepSeek API**: 用于文本分析和元数据提取
- **Kimi API**: 用于视觉模型（页眉页脚识别）
- **OpenAlex**: 免费，建议间隔1-2秒
- **Crossref**: 免费，需要邮箱标识
- **Semantic Scholar**: 免费，速率限制宽松

### 处理参数
- **OCR阈值**: 100字符（低于此值触发OCR）
- **首次扫描页数**: 前3页
- **最大工作线程**: 3（API并发数）
- **保存间隔**: 每10个文件保存一次
- **重试次数**: 3次
- **重试间隔**: 2^attempt秒（指数退避）
